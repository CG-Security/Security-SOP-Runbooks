# Contributing Guidelines

Thank you for your interest in contributing to Security-SOP-Runbooks. This project documents repeatable operational security workflows and aims to maintain a consistent structure across all SOPs.

## Contribution Expectations

All SOPs must:
- Use the established format (Purpose, Scope, Preconditions, Roles, Procedure, Evidence Requirements, Success Criteria)
- Remain generic and safe for public use
- Avoid organization-specific references
- Use clear, direct, procedural language
- Focus on operational steps rather than policy statements

## How to Contribute

### Improving Existing SOPs
If you want to clarify a step, add missing context, or correct inaccuracies:
1. Open an issue describing the proposed improvement.
2. Submit a pull request with the updated content.

### Adding New SOPs
If adding a new SOP:
1. Open an issue proposing the topic.
2. Follow the standard SOP structure.
3. Submit a pull request for review.
